# AI Agent 在 CDP 中的应用实践

## 背景

传统 CDP 的标签运算、受众分群、数据清洗等环节需要大量人工规则配置。AI Agent 可以自动识别数据模式、生成标签规则、优化分群策略。

## 架构思路

### 三层 AI 能力

```
┌─────────────────────────────────────────┐
│  应用层：智能问答 / 自动标签 / 异常检测    │
├─────────────────────────────────────────┤
│  Agent 层：任务编排 / 工具调用 / 记忆管理  │
├─────────────────────────────────────────┤
│  基础层：LLM / Embedding / Vector DB     │
└─────────────────────────────────────────┘
```

### 核心 Agent 角色

1. **数据理解 Agent**：自动分析数据分布，推荐标签体系
2. **受众分群 Agent**：自然语言输入 → 自动生成 SQL/规则
3. **异常检测 Agent**：监控数据质量，自动告警

## 经验教训

- Agent 的幻觉率在标签生成场景可接受（5% 以下），但数据计算场景必须严格校验
- 需要建立"人审 + 自动执行"的双轨机制，Agent 生成规则 → 人工确认 → 自动生效
